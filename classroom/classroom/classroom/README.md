# Data Processor / Data Pipeline

### Цель

Реализовать модульный пайплайн обработки коллекции товаров (фильтрация → преобразование → агрегация → сортировка) на чистом JavaScript. Демонстрирует композицию функций и работу с `map/filter/reduce`.

---

## Структура проекта

```
project/
  package.json
  README.md
  src/
    data.js
    pipeline.js        # реализация pipe(...fns)
    filters.js
    mappers.js
    reducers.js
    processors.js      # готовые пайплайны
    index.js            # запуск и вывод
```

---

## Исходные данные

`data.js` экспортирует массив объектов `{ id, name, price, category, inStock }` (минимум 10 элементов).

---

### Задачи

1. `pipe`

   - Сигнатура: `pipe(...functions)`
   - Описание: возвращает функцию, последовательно применяющую переданные функции к данным.
   - Требование: реализовать через `reduce`; не мутировать данные.

2. Фильтры (`filters.js`)

   - `filterByCategory(category)` — возвращает функцию-фильтр по категории.
   - `filterInStock` — возвращает только товары с `inStock === true`.
   - Использовать `filter` и чистые функции.

3. Мапперы (`mappers.js`)

   - `mapToNames` — возвращает массив имён.
   - `mapToPriceWithTax(tax)` — возвращает функцию, добавляющую налог к цене (не мутируя объекты).

4. Агрегаторы (`reducers.js`)

   - `sumPrices` — сумма цен.
   - `groupByCategory` — группировка по `category`, возвращает объект.
   - Использовать `reduce`.

5. Сортировки (`processors.js`)

   - `sortByPriceAsc`, `sortByPriceDesc` — возвращают новый отсортированный массив (`slice` + `sort`).

6. Собрать минимум 3 пайплайна (`processors.js`) с использованием `pipe` и готовых функций.

7. `index.js` — импортировать данные и пайплайны, вывести результаты в консоль для демонстрации.

---

### Технические требования

- Использовать `const`/`let` и стрелочные функции.
- Чистые функции; не мутировать входные данные.
- Активно использовать `map`, `filter`, `reduce`, spread/rest.
- Добавить примеры использования и инструкции в `README.md`.

## Примеры использования

Реализация модулей находится в папке `src/` — основные файлы:

- `src/data.js` — тестовые данные
- `src/pipeline.js` — `pipe(...fns)`
- `src/filters.js`, `src/mappers.js`, `src/reducers.js` — вспомогательные функции
- `src/processors.js` — сортировки и готовые пайплайны
- `src/index.js` — демонстрация работы

Запуск примера в консоли:

```bash
npm start
```

Ожидается вывод нескольких демонстрационных результатов: список имён товаров в наличии, электроника с налогом, группировка по категориям и суммарная стоимость в наличии.
